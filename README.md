# ЛАБОРАТОРНАЯ РАБОТА №6 (Fetch API, сборка и деплой)

**Задача:** модернизация веб-приложения путем перехода с XMLHttpRequest на fetch с использованием async/await, а также интеграция сборщика Vite и настройка сервера NestJS для раздачи фронтенда в качестве статики с целью устранения проблем с CORS

**Сделано:**
- Выполнен рефакторинг API-клиента: устаревший интерфейс XMLHttpRequest заменен на стандарт Fetch API. Использование конструкций async/await и Promise позволило упростить структуру асинхронного кода, повысить его читаемость и унифицировать механизм обработки исключений
- Разработан интерактивный обработчик формы «Сохранить», обеспечивающий динамическую отправку POST и PATCH запросов без перезагрузки страницы, а также реализован механизм предзаполнения полей данными текущей записи при редактировании через GET запрос
- В проект интегрирован сборщик Vite, настроен конфигурационный файл vite.config.js и определены скрипты жизненного цикла разработки. Код фронтенда переведен на формат минифицированного production-бандла, что обеспечило высокую производительность клиентского приложения
- Реализована раздача статических файлов фронтенда серверным приложением, что позволило объединить клиентскую и серверную части на одном домене (http://localhost:3000). Это архитектурное решение полностью устранило проблему блокировки сетевых запросов политикой безопасности CORS
